# decisions/ — 架構決策記錄（ADR）

記錄重要的技術決策與取捨，供未來開發者理解「為什麼這樣做」。

## 何時建立

- 選定某個重要套件或框架（例如「為什麼用 Prisma 而非 TypeORM」）
- 架構層級的方向選擇（例如「為什麼採用 App Router 而非 Pages Router」）
- 跨專案的協定變更（例如 API 回應格式、錯誤碼標準）

## 命名

```
{YYYYMMDD}-{NNN}-{kebab-case-title}.md
```

## 建議內容結構

```markdown
# ADR-{NNN}: {決策標題}

> 日期: {YYYY-MM-DD}
> 狀態: 提案中 / 已採納 / 已棄用 / 已被取代

## 背景
## 決策
## 理由
## 後果（正面 / 負面）
## 替代方案
```

## 與其他文件的差異

| 文件類型 | 定位 |
| --- | --- |
| `plans/` | 功能要做什麼、怎麼做（執行導向） |
| `specs/` | 技術規格（檔案層級） |
| `knowledge/` | 可複用的知識（從多個來源提煉） |
| `decisions/` | **不可逆或高影響的決策**，記錄選擇的脈絡與理由 |
